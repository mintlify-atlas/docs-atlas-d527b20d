The silence version provides a lightweight Discord Rich Presence integration that shows only basic status without tracking specific activities.

## What it tracks

The silence version displays:

- **Program status**: Shows that MobaXterm is running
- **Timestamp**: How long the application has been open
- **Logo**: MobaXterm branding

That's it. No process monitoring, no activity detection, no frequent updates.

<Info>
This version uses significantly fewer system resources than the complete version since it doesn't monitor running processes.
</Info>

## Core implementation

The silence version is much simpler than the complete version. Here's the entire Discord thread implementation:

```cpp silence.cpp
DWORD WINAPI DiscordThread(LPVOID lpParam) {
    DiscordEventHandlers handlers;
    memset(&handlers, 0, sizeof(handlers));
    handlers.ready = HandleDiscordReady;

    Discord_Initialize("1347921816777392244", &handlers, 1, NULL);

    DiscordRichPresence presence;
    memset(&presence, 0, sizeof(presence));
    presence.state = "";
    presence.details = "";
    presence.startTimestamp = time(0);
    presence.largeImageKey = "mobaxterm-logo";

    Discord_UpdatePresence(&presence);

    while (g_Running) {
        Discord_RunCallbacks();
        Sleep(500);
    }

    Discord_Shutdown();
    return 0;
}
```

### Key differences from complete version

Your Discord profile will show:

- **Large image**: MobaXterm logo
- **Timestamp**: Elapsed time since MobaXterm started
- **State**: Empty (clean appearance)
- **Details**: Empty (clean appearance)

<Note>
The silence version intentionally leaves state and details empty to maintain a minimal presence.
</Note>

## When to use this version

Choose the silence version if you:

- Want Discord integration without activity details
- Prefer minimal system resource usage
- Don't want others to see specific activities
- Value privacy over detailed status
- Use MobaXterm on resource-constrained systems

<Info>
The silence version is ideal for users who want Discord to show they're using MobaXterm without revealing what they're doing in the application.
</Info>

## Switching between versions

Both versions use the same DLL injection approach:

1. Replace `MobaXterm.exe` with the version-specific build
2. Ensure `discord-rpc.dll` and `final.dll` are in the installation directory
3. Restart MobaXterm

<Note>
You can switch between versions at any time by swapping the executable. No configuration changes needed.
</Note>

## Privacy considerations

The silence version offers better privacy:

- No indication of text editing activity
- No terminal session information
- No Bash usage detection
- Others only see "Using MobaXterm"

Compare this to the complete version which broadcasts:
- When you're editing files
- When you have terminal sessions open
- Whether you're running local Bash

If you're working on sensitive projects or prefer to keep your workflow private, the silence version is the better choice.
